## Decisions & Flags

### Tailwind `@layer` removal
The original `@layer base { }` and `@layer components { }` wrappers were removed. Cascade is now controlled by import order (CUBE model) rather than Tailwind's layer system. Tailwind utilities still function because they're generated by the Tailwind plugin at the utility cascade level.

### `@apply` concern mixing
`.hash-chip` and `.hash-filter-btn` use `@apply` directives that mix layout and visual properties. These were kept intact in blocks.css since splitting them would break Tailwind compilation. Documented as a known concern-mix.

### `@keyframes` placement
All `@keyframes` are in blocks.css alongside the components they animate, per the instruction: "go in the file that owns the element they animate."

### `.ghost` in utilities, not blocks
`.ghost` / `.ghost.awake` was placed in utilities (visibility toggle is a single-purpose job) rather than blocks, even though it has two properties (opacity + pointer-events). The pair always changes together as a single intent.

### Collapse pattern split across 3 files
- `compositions.css`: `display: grid; grid-template-rows: 0fr` (geometry)
- `blocks.css`: `transition: grid-template-rows` (visual treatment)
- `exceptions.css`: `[data-expanded='true'] { grid-template-rows: 1fr }` (data-driven state)

### Raw pixel extractions
Component-specific pixel values were extracted to tokens and replaced with `calc(var(--spacing) * N)` expressions or dedicated tokens. Border-radius `6px` was subsumed into `var(--radius-md)` (8px â†’ close enough).
